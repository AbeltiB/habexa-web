# habexa-web — CLAUDE.md

## What this repo is

The user-facing web application and PWA for Habexa. This is the primary product — what users open when they go to `habexa.com`. It is a mobile-first responsive web app that installs as a PWA on Android devices.

**URL:** `https://habexa.com`
**Platform:** Vercel
**Primary viewport:** 390px mobile (iPhone 14 Pro width) — design for this first, then desktop

## Routing Logic

```typescript
// middleware.ts
// Rule 1: Unauthenticated user → always redirect to /login
// Rule 2: Authenticated but no displayName → redirect to /onboarding
// Rule 3: /login and /verify → redirect to / if already authenticated
// Rule 4: All /app/* routes require auth (handled by (app)/layout.tsx guard too)
```

No route under `(app)/` should be reachable without a valid token. Both middleware and the layout component check — defense in depth.

## API Client

The API client in `lib/api.ts` is the only place that calls `habexa-api`. No component should ever call `fetch()` directly.


## SWR Data Fetching

Use SWR for all server data. Never use `useEffect + fetch`. The SWR key is always the API path.

```typescript
// Standard pattern for all data hooks:
// lib/hooks/usePortfolio.ts
import useSWR from 'swr'
import { api } from '../api'
import type { PaperAccountSummary } from '@habexa/sdk'

export function usePortfolio() {
  const { data, error, isLoading, mutate } = useSWR<PaperAccountSummary>(
    '/api/paper/account',
    () => api.paper.account(),
    { refreshInterval: 5 * 60 * 1000 }  // refresh every 5 min (not real-time)
  )

  return { portfolio: data, error, isLoading, refresh: mutate }
}
```

SWR cache keys must match API paths exactly — this makes debugging predictable.

Refresh intervals:
- Market prices: every 5 minutes (prices only update once/day anyway)
- Portfolio: every 5 minutes
- Modules: no auto-refresh (static content)
- Leaderboard: every 15 minutes

## i18n

```typescript
// Usage in any component:
import { useLanguage } from '@/components/providers/LanguageProvider'
import { useTranslation } from '@/lib/i18n'

function MyComponent() {
  const { language } = useLanguage()
  const { t } = useTranslation(language)

  return <h1>{t('dashboard.portfolio_value')}</h1>
}
```

Rules:
- Every user-facing string must have both `am` and `en` keys in `locales/`
- Never hardcode display strings — only use `t()` keys
- Key format: `section.subsection.key`
- If a translation key is missing: fall back to English, log a warning in dev

## PWA Configuration

```typescript
// next.config.ts
import withPWA from '@ducanh2912/next-pwa'

export default withPWA({
  dest: 'public',
  cacheOnFrontEndNav: true,
  aggressiveFrontEndNavCaching: true,
  reloadOnOnline: true,
  disable: process.env.NODE_ENV === 'development',
  workboxOptions: {
    disableDevLogs: true,
    runtimeCaching: [
      {
        // Cache API responses for offline viewing
        urlPattern: /^https:\/\/api\.habexa\.com\/api\/modules/,
        handler: 'NetworkFirst',
        options: { cacheName: 'api-modules', expiration: { maxAgeSeconds: 86400 } }
      },
      {
        // Cache Cloudflare Stream thumbnails
        urlPattern: /^https:\/\/.*cloudflarestream\.com/,
        handler: 'CacheFirst',
        options: { cacheName: 'video-thumbnails', expiration: { maxAgeSeconds: 604800 } }
      }
    ]
  }
})
```

What works offline (service worker cached):
- Previously viewed module articles
- Module list (stale)
- Portfolio last known state
- Static assets (fonts, icons, CSS)

What does NOT work offline:
- Video playback (stream only)
- Paper trading (requires live prices)
- Leaderboard (requires live data)

## Component Conventions

**Naming:**
- Pages: `page.tsx` (Next.js convention)
- Client components: PascalCase, named export, add `'use client'` only when needed
- Hooks: `use` prefix, live in `lib/hooks/`
- No default exports except pages and layouts

**State management:**
- Server state: SWR hooks
- UI state: `useState` / `useReducer` in component
- Global state: Context only for auth and language (two things, not more)
- No Redux, no Zustand — not needed at this scale

**Loading states:**
- Every data-dependent component must handle `isLoading` with a skeleton, not a spinner
- Use `shadcn/ui Skeleton` component for skeletons
- Never show a blank screen while loading

**Error states:**
- Every SWR hook consumer must handle `error`
- Show an inline error message with a retry button
- Never let errors silently fail

**Premium gating:**
```typescript
// Standard pattern for premium-gated features
import { PremiumGate } from '@/components/shared/PremiumGate'

function PremiumFeature() {
  const { user } = useAuth()
  if (!user?.isPremium) return <PremiumGate feature="leaderboard" />
  return <ActualFeature />
}
```

```bash
# Web Push

NEXT_PUBLIC_PLAUSIBLE_DOMAIN=habexa.com

NEXT_PUBLIC_APP_URL=https://habexa.com
```

Only `NEXT_PUBLIC_` prefixed variables are exposed to the browser. Never put secrets here.

## Design System

- Font: Geist Sans (Next.js default) — clean, works in both Amharic and Latin scripts
- Colors: defined in `tailwind.config.ts` — primary green (#16a34a), dark background (#0a0a0a)
- Mobile-first: design at 390px, add md: breakpoints for tablet/desktop
- Amharic text: increase line-height to 1.8 for readability (Ethiopic script needs space)
- Numbers (ETB): always use `formatETB()` from `@habexa/sdk` — never format manually
